### 1. ランディング `/`

**目的：** 未ログインユーザーへのアプリ紹介・サインアップ誘導

**アクセス制御：**
- ログイン済みの場合は `/today` へ自動リダイレクト

**主要 UI 要素：**

| 要素 | 説明 |
|------|------|
| ヒーローセクション | 「おでかけを、もっと楽しく。」のキャッチコピー |
| グラデーション背景 | ダークグリーン系グラデーション |
| 無料で始める ボタン | `/auth/sign-up` へ遷移 |
| ログイン リンク | `/auth/sign-in` へ遷移 |
| スクロールインジケーター | 下矢印アニメーション |

**遷移先：**

```
無料で始める → /auth/sign-up
ログイン     → /auth/sign-in
（ログイン済み）→ /today（自動）
```

### 2. 今日 `/today`

**目的：** 今日・明後日までの予定とイベントをまとめて表示するホーム画面

**アクセス制御：** ログイン必須

**主要 UI 要素：**

| 要素 | 説明 |
|------|------|
| グリーティング | 時間帯に応じた挨拶（おはようございます / こんにちは / こんばんは）+ ユーザー名 |
| 今日の予定リスト | マイ予定（`personal_events`）とスケジュール済みイベント（`schedules`）を時刻順に表示 |
| チェックボックス | 今日の予定を完了済みとしてマーク（ UI 状態のみ、DB には保存しない） |
| 明日・明後日のイベント | 今後 2 日間に開催される公開イベントをサジェスト |
| カレンダーに追加 ボタン | スケジュールに登録（`POST /api/v1/schedules`） |
| マイ予定追加 ボタン（+） | PersonalEventModal を開く |
| PersonalEventModal | タイトル・メモ・日付・開始終了時間・場所・URL を入力するモーダル |

**表示ルール：**
- マイ予定とスケジュール済みイベントは `start_time` / `start_at` で昇順ソート
- 時刻なしの予定は「終日」として最後尾に表示
- チェック済み予定はフェードアウト表示（透明度 45%）

**遷移先：**

```
イベントカード → /events/[id]
```

---

### 3. イベント一覧 `/events`

**目的：** 公開イベントの一覧表示・検索・絞り込み

**アクセス制御：** 不要（お気に入り操作のみログイン必須）

**主要 UI 要素：**

| 要素 | 説明 |
|------|------|
| 検索バー | イベント名・場所・エリアでフロント側フィルタリング |
| 絞り込みボタン | FilterDrawer を開く |
| タブ（すべて / お気に入り） | お気に入りタブはログイン時のみ機能 |
| アクティブフィルターチップ | 選択中のカテゴリ・エリア・タグを表示。クリックで個別解除 |
| スケルトンローディング | データ取得中は 10 枚のスケルトンカードを表示 |
| イベントカードグリッド | `auto-fill / minmax(255px, 1fr)` のグリッドレイアウト |
| 終了済みを表示 トグル | 過去 1 年以内のイベントを表示切替 |
| ページドロップダウン | 1 ページ 12 件。総ページ数が 2 以上の場合に表示 |
| ページネーション | ページ下部に表示 |

**FilterDrawer の絞り込み項目：**

| 項目 | 内容 |
|------|------|
| エリア | 福島県全 59 市町村をグループ別 react-select で選択（複数可） |
| カテゴリ | テクノロジー / 音楽 / スポーツ / 自然・アウトドア / 食・グルメ / 文化・伝統 / ファミリー / 教育 / 祭り・イベント / アート / その他 |
| タグ | イベントに付与されたタグで絞り込み |

**遷移先：**

```
イベントカード → /events/[id]
```

### 4. イベント詳細 `/events/[id]`

**目的：** イベントの詳細情報を表示し、お気に入り・スケジュール追加を行う

**アクセス制御：** 不要（お気に入り・スケジュール操作のみログイン必須）

**主要 UI 要素：**

| 要素 | 説明 |
|------|------|
| イベント画像 | 上部全幅（`image_url` が未設定の場合はカテゴリ色の背景） |
| カテゴリバッジ | カテゴリ名とアイコン（カテゴリ別に色を変える） |
| タイトル | イベント名 |
| 日時 | 開始・終了日時（日本語フォーマット） |
| 天気バッジ | 開始日時の天気予報（WeatherBadge コンポーネント） |
| 場所 | 会場名とマップ（Leaflet でピン表示） |
| 説明文 | イベント概要 |
| 参加者数・定員 | `accepted` / `limit` |
| お気に入りボタン | ハートアイコン。ログイン済みのみ機能 |
| カレンダーに追加ボタン | スケジュールに登録。ログイン済みのみ機能 |
| 公式サイトを見る | 外部リンク（`event_url`） |

**遷移先：**

```
← 戻る → /events（ブラウザバック）
公式サイトを見る → 外部サイト（新規タブ）
```

### 5. 個人予定 `/personal-events`

**目的：** ユーザー独自のマイ予定の一覧・追加・編集・削除

**アクセス制御：** ログイン必須（未ログインはデータ取得しない）

**主要 UI 要素：**

| 要素 | 説明 |
|------|------|
| 今日・未来グループ | 日付ごとにグループ化して表示。今日は「今日」と明示 |
| 過去の予定 | アコーディオンで折りたたみ表示 |
| 予定行 | 時刻・タイトル・場所・メモを表示。ホバーで編集ボタン出現 |
| 編集ボタン（ペン） | PersonalEventModal を編集モードで開く |
| ＋ ボタン（FAB） | PersonalEventModal を新規作成モードで開く |
| PersonalEventModal | タイトル・メモ・日付・開始終了時間・場所・URL を入力 |
| 空状態イラスト | 予定が 0 件の場合は undraw のイラストを表示 |

**CRUD 操作：**

| 操作 | API |
|------|-----|
| 一覧取得 | `GET /api/v1/personal_events` |
| 新規作成 | `POST /api/v1/personal_events` |
| 更新 | `PUT /api/v1/personal_events/:id` |
| 削除 | `DELETE /api/v1/personal_events/:id` |

### 8. マップ制覇 `/conquer`

**目的：** 福島県内の市町村への訪問を記録し、制覇状況をマップで確認

**アクセス制御：** ログイン必須

**主要 UI 要素：**

| 要素 | 説明 |
|------|------|
| 福島県 SVG マップ | 市町村ごとに塗り分け。制覇済みは色付き、未制覇はグレー |
| 訪問記録モーダル | 市町村クリックで開く。写真・メモ・日付を記録 |
| 制覇確定ボタン | 訪問記録が 1 件以上の市町村に「制覇する」ボタンを表示 |
| ユーザー統計 | 制覇数 / 総市町村数（59）を表示 |
| リセットボタン | 訪問記録・制覇記録をすべて削除 |

**API：**

| 操作 | API |
|------|-----|
| 訪問記録一覧 | `GET /api/v1/visit_records` |
| 訪問記録追加 | `POST /api/v1/visit_records` |
| 訪問記録更新 | `PATCH /api/v1/visit_records/:id` |
| 訪問記録削除 | `DELETE /api/v1/visit_records/:id` |
| 制覇済み一覧 | `GET /api/v1/region_conquests` |
| 制覇登録 | `POST /api/v1/region_conquests` |
| 制覇リセット | `DELETE /api/v1/region_conquests/destroy_all` |
| 未制覇・訪問済み市町村 | `GET /api/v1/conquer/pending_confirmations` |

### 17. 管理：AI インポート `/admin/ai-import`

**目的：** 画像または PDF からイベント情報を AI（Gemini）で抽出し、DB に登録

**主要 UI 要素：**

| 要素 | 説明 |
|------|------|
| 画像アップロード | チラシ画像（PNG / JPG）を選択 |
| PDF アップロード | チラシ PDF を選択 |
| 抽出ボタン | `POST /api/v1/admin/ai_import` にファイルを送信 |
| 抽出結果プレビュー | AI が抽出したイベント情報をフォームで確認・修正 |
| 登録ボタン | 確認済みのイベントを一括登録 |

**処理フロー：**

```
チラシ画像 / PDF をアップロード
  ↓
POST /api/v1/admin/ai_import（Gemini API でテキスト抽出）
  ↓
抽出されたイベント情報をプレビュー表示
  ↓
ユーザーが内容を確認・修正
  ↓
登録ボタン → POST /api/v1/events（各イベントを個別に登録）
  ↓
登録完了 toast → /admin/events へ遷移
```
