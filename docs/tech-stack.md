# 技術スタック

## フロントエンド

| カテゴリ | 使用技術 | バージョン | 選定理由 |
|----------|----------|-----------|---------|
| 言語 | TypeScript | 5.x | 型安全で開発時のミスを減らせる |
| フレームワーク | Next.js（App Router） | 14.x | React ベースの SSR フレームワーク。SEO・パフォーマンスに優れる |
| スタイリング | Tailwind CSS | 3.x | ユーティリティクラスで素早く UI を組める |
| カレンダー | FullCalendar | 6.x | React 対応のカレンダーライブラリ。週・月表示が標準搭載 |
| HTTP クライアント | fetch API（標準） | - | Next.js の Server Components に組み込まれており、追加ライブラリ不要 |
| 状態管理 | React useState / Context | - | MVP レベルであれば外部ライブラリ不要 |

## バックエンド

| カテゴリ | 使用技術 | バージョン | 選定理由 |
|----------|----------|-----------|---------|
| 言語 | Ruby | 3.3.x | Rails との親和性が最も高い |
| フレームワーク | Ruby on Rails（API モード） | 7.2.x | REST API サーバーとして広く使われている |
| ORM | ActiveRecord | - | Rails に標準搭載。マイグレーション・アソシエーションが使いやすい |
| 認証 | Devise | 4.x | Rails の事実上の標準認証ライブラリ |
| CORS 対応 | rack-cors | 2.x | フロントエンド（Next.js）からのリクエストを許可するために必要 |
| JSON 生成 | jbuilder / ActiveModel::Serializers | - | API レスポンスの JSON 形式を定義する |

## データベース

| カテゴリ | 使用技術 | バージョン | 選定理由 |
|----------|----------|-----------|---------|
| データベース | MySQL | 8.0 | 日本で広く使われているリレーショナル DB |
| ローカル環境 | Docker + docker-compose | - | 環境構築を簡単にするためにコンテナで動かす |

## インフラ・デプロイ

| カテゴリ | 使用技術 | 選定理由 |
|----------|----------|---------|
| クラウド | AWS | 国内シェア No.1。転職市場での需要が最も高い |
| サーバー | EC2 | Rails API サーバーをホスティング |
| データベース | RDS（MySQL） | マネージド DB。バックアップ・可用性を AWS が管理してくれる |
| Webサーバー | nginx | 静的ファイル配信と API プロキシを兼任 |
| IaC | Terraform | インフラをコードで管理（TaskManagement アプリと同様） |

## 共通

| カテゴリ | 使用技術 | 選定理由 |
|----------|----------|---------|
| バージョン管理 | Git / GitHub | 業界標準のコード管理ツール |
| コンテナ | Docker | ローカル開発の MySQL 起動に使用 |

## ポート番号（固定・変更禁止）

| サーバー | ポート | 備考 |
|---------|--------|------|
| Next.js（フロントエンド） | **3000** | `npm run dev` のデフォルト |
| Rails（バックエンド） | **8080** | `rails s -p 8080` で起動 |
| MySQL（データベース） | **3306** | docker-compose で起動 |

> **なぜポートを固定するか：** Next.js の API プロキシ設定（`next.config.js`）と Rails の CORS 設定が特定のポートを前提にしているため。別ポートで動かすと通信が失敗する。

## 採用しない技術と理由

| 技術 | 採用しない理由 |
|------|--------------|
| GraphQL | REST API で十分。学習コストが高い |
| Redux | Context API で十分な規模 |
| Webスクレイピング | 法的リスクがあるため。Connpass 公式 API のみ使用 |
| WebSocket | リアルタイム通信は MVP 後に検討 |
